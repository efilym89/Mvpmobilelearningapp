# Project Guidelines

Этот файл делает правила из `idea.md`, `vision.md` и `conventions.md` рабочими для текущего репозитория.

## Источники истины

- `vision.md` определяет целевую архитектуру и MVP-ограничения.
- `conventions.md` определяет правила выбора решений.
- `idea.md` определяет продуктовую цель и базовый сценарий.

## Правила для изменений

- Реализуем только сценарии, которые помогают проверить MVP-идею.
- Предпочитаем простые решения и короткий путь к рабочему результату.
- Не добавляем новые сервисы, абстракции и библиотеки без явной пользы для MVP.
- Не переносим backend-логику во frontend.
- Не создаём shared-пакеты заранее, если повторного использования ещё нет.
- Конфигурацию держим через переменные окружения.
- Логи и тесты добавляем только там, где они реально защищают критичный сценарий.

## Правило для текущего репозитория

Пока основная реализация живёт в `src/` как web-прототип. Новые изменения в этом каталоге должны:

- поддерживать ключевые сценарии обучения и тестирования;
- не превращать прототип в сложную постоянную архитектуру;
- готовить проект к переходу в `apps/mobile`, `apps/admin` и `apps/backend`.
