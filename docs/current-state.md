# Текущее состояние и переход к vision.md

## Что есть сейчас

Репозиторий содержит один web-прототип на `Vite/React`, который помогает быстро проверять пользовательские сценарии MVP:

- вход пользователя;
- каталог курсов;
- просмотр курса и урока;
- прохождение теста;
- просмотр результата;
- базовый административный экран.

## Что зафиксировано как целевое состояние

Согласно `vision.md`, проект должен развиваться как простой `pnpm`-monorepo со следующими частями:

- мобильное приложение для сотрудника;
- отдельная административная панель;
- единый backend-монолит;
- общие контракты;
- минимальная инфраструктура;
- простая документационная база.

## Как применять это к текущему репозиторию

- не раздувать существующий `src/` новыми платформенными абстракциями;
- использовать текущий прототип только как быстрый способ проверки сценариев;
- новые доменные решения описывать через общие типы и контракты;
- инфраструктурные и архитектурные шаги делать постепенно, без пустых сложных заготовок.

## Практический переход

1. Сохраняем текущий прототип работоспособным.
2. Закрепляем новые правила в документации и доменных модулях.
3. Переносим функциональность в `apps/mobile`, `apps/admin` и `apps/backend` по мере появления реальной реализации.
4. Выносим действительно общие контракты в `packages/shared-contracts`, когда ими начнут пользоваться несколько приложений.
